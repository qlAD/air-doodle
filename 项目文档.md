# AI 手势涂鸦工坊（Air Doodle Studio）项目文档

## 1 项目概述

### 1.1 项目简介

《AI 手势涂鸦工坊》是一款**基于浏览器摄像头、前端 AI 识别、Next.js 全栈架构**的软硬件结合交互式创意项目。项目仅依托电脑/手机内置摄像头作为唯一硬件采集设备，通过 WebRTC 实时获取视频流，结合 MediaPipe WASM 轻量化手部 AI 模型，在浏览器本地完成手势识别，实现隔空无接触涂鸦绘画。

本项目摒弃传统鼠标、数位板、触屏的交互方式，实现纯体感手势绘画，并基于 Next.js 全栈能力，实现画作云端存储、在线画廊展示、作品社区互动、模板临摹、体感小游戏、社交分享等全流程功能。项目全程浏览器运行，跨设备兼容、无需安装客户端，兼具趣味性、工程性与创新性，可用于课程设计、全栈开发实训、科创作品展示。

### 1.2 项目开发背景

传统手势涂鸦方案多基于 Python 本地桌面开发，存在明显短板：依赖本地运行环境、仅支持单机使用、无法云端存储、无法在线分享、扩展性差。同时传统电子白板交互方式单一，人机交互体验生硬。

随着 WebAI、WebRTC、WASM 技术成熟，浏览器已具备离线 AI 计算、实时视频处理的能力。基于 Next.js 全栈框架重构手势涂鸦项目，能够实现**前端 AI 识别 + 后端数据持久化**的完整 Web 应用，解决传统本地项目的所有痛点，形成标准化、可部署、可迭代的现代化 Web 项目。

此外，现有手势涂鸦类项目普遍存在**玩法单一、社交属性缺失、新手上手门槛高、缺少分享激励**等核心短板——用户往往玩 5 分钟便流失。本项目针对这些痛点进行系统性升级，从单一"手势画画工具"转型为**轻量化体感互动娱乐 Web 应用**，覆盖自由涂鸦、模板临摹、体感小游戏、双人同屏协作、作品社区分享等多元玩法，大幅提升用户留存与自发传播能力。

### 1.3 项目创新点

1. **轻量化软硬件结合**：仅使用设备自带摄像头为唯一硬件，依托 Web 标准实现图像采集，零硬件成本、跨设备适配，软硬件交互闭环完整。

2. **前端离线 AI 计算**：基于 MediaPipe WASM 在浏览器本地完成手势识别，视频流不上传服务器，低延迟、保护隐私、节省后端算力。

3. **Next.js 全栈一体化**：一套框架实现客户端交互、服务端渲染、API 接口、数据库存储、静态部署，工程结构规范。

4. **功能多元化交互**：支持手势绘画、擦除、换色、暂停、清空等多维度体感操作，交互逻辑人性化，误触率低。

5. **数据持久化与在线展示**：支持画作在线保存、画廊分页展示、作品管理，突破传统单机涂鸦无留存、无分享的短板。

6. **零门槛临摹创作体系**：内置海量分层临摹模板（卡通人物、小动物、简笔画、节日素材等），半透明底图覆盖画布，绘画小白沿轮廓描线即可快速产出完整画作，大幅降低创作门槛；完成临摹自动弹出成就勋章。

7. **多模式体感娱乐闭环**：融合自由涂鸦、模板临摹、手势体感小游戏（消消画、抓星星、限时猜画），不止单一画画工具，覆盖手绘小白与创意玩家全人群，有效拉长用户游玩时长。

8. **双人多手协同交互**：基于 MediaPipe 多手检测能力，支持同一设备同时识别双手/双人手掌，实现情侣、亲子同屏隔空共创绘画与对战小游戏，线下互动性极强。

9. **视听一体化体感反馈**：配套动态特效笔刷（星光笔、彩虹笔、烟花笔、水彩晕染、荧光霓虹、像素方块）、全屏烟花动画、场景音效、移动端触觉震动，全方位强化无接触交互的沉浸感与氛围感。

10. **完整社交分享传播链路**：作品社区支持点赞、收藏、评论、复刻临摹、热门排行榜；一键生成绘画过程短视频与分享海报（自动添加 Air Doodle 水印、趣味边框、文案模板），支持下载图片/视频、复制分享链接，自带自发传播属性。

11. **成长激励成就系统**：连续打卡勋章、创作数量成就、画笔解锁、小游戏通关等层级化激励体系，解锁专属限定素材，提升用户复访频率与长期粘性；节日限时主题活动定期更新内容。

### 1.4 运行环境

#### 硬件环境

搭载摄像头的 PC、笔记本、智能手机，支持 WebRTC 标准的多媒体设备。

#### 软件环境

操作系统：Windows / macOS / Android / iOS；浏览器：Chrome、Edge、Safari 最新版；开发环境：Node.js 18 +、Next.js 15

## 2 系统需求分析

### 2.1 功能需求

#### 2.1.1 基础交互功能

- 摄像头实时预览、镜像翻转适配体感操作；

- 食指单点隔空绘画，轨迹平滑流畅；

- 握拳手势暂停绘画，避免误操作；

- 手掌摊开触发橡皮擦功能，局部擦除涂鸦；双手摊开一键雾化擦除；

- OK 手势切换画笔颜色（红/蓝/绿），不同 OK 手势组合切换特效画笔；

- 双手长按握拳清空整张画布，清空时全屏烟花动画反馈；

- 动态特效画笔：星光笔、彩虹笔、烟花笔、水彩晕染、荧光霓虹、像素方块，双手比心开启爱心画笔；

- 动态背景画布：星空、云朵、海浪、渐变、节日动态背景，手势一键切换。

#### 2.1.2 模板临摹功能

- 内置海量分层临摹模板库（卡通人物、小动物、星空、简笔画、节日素材）；

- 半透明底图铺在画布底层，用户隔空沿轮廓描线；

- 握拳切换上一张模板，摊手切换下一张模板，OK 键调整模板透明度；

- 分难度等级：幼儿简笔画 / 二次元插画 / 风景素描；

- 完成临摹自动弹出「临摹完成勋章」；

- 支持将社区他人作品一键设为临摹模板（复刻临摹）。

#### 2.1.3 体感小游戏功能

复用手部关键点识别逻辑，内置轻量化小游戏，一键切换模式：

- **手势消消画**：屏幕散落彩色圆点，食指连线相同颜色消除，计分闯关；

- **隔空抓星星**：画面随机生成星星，手掌抓取收集，限时计分排行；

- **手势猜谜画画**：系统给出词语（云朵、小猫、汽车），限时手势绘画，AI 自动打分。

#### 2.1.4 动态贴纸与隔空贴图

- 比耶手势呼出贴纸面板（表情包、云朵、花朵、节日装饰）；

- 单指拖动贴纸摆放，握拳缩放大小，手掌擦掉贴纸；

- 零绘画基础也能快速拼出好看作品，小白零门槛出片。

#### 2.1.5 社交互动功能

- 作品社区：公开投稿、点赞、收藏、评论；

- 复刻临摹：一键将他人画作设为临摹模板；

- 作品排行榜：今日热门、本周最佳临摹、创意榜单；

- 一键生成分享海报：自动添加 Air Doodle 水印、趣味边框、文案模板（"我用隔空手势画出XX"）；

- 绘画全过程自动录屏（前端 MediaRecorder），生成 3-10 秒短视频，支持下载与多平台分享；

- 画作可选私密保存（仅自己可见）或公开投稿社区。

#### 2.1.6 用户成就系统

- 连续打卡 3/7/30 天勋章；

- 创作成就：完成 N 幅画作、解锁全部笔刷、通关小游戏等；

- 成就解锁专属限定画笔、稀有贴纸模板，形成长期游玩激励。

#### 2.1.7 全栈业务功能

- 画布画作导出为图片，上传后端服务器保存；

- 服务端数据库存储作品信息与图片路径；

- 在线画廊页面 SSR 渲染所有作品，分页展示；

- 支持作品删除、详情预览；

- 用户信息管理、成就数据存储；

- 页面响应式适配电脑、手机端。

#### 2.1.8 其他特色功能

- **手势文字涂鸦**：隔空写出的线条自动规整为文字，支持手写签名、隔空写祝福；

- **双人同屏手势绘画**：同一设备双人同时识别左右手，两人隔空一起画画、对战小游戏；

- **节日限定主题活动**：端午、中秋、圣诞、新年自动上线限定模板、笔刷、活动榜单；

- **AI 一键美化画作**：前端轻量图像滤镜，画完一键动漫化、油画、素描、胶片滤镜；

- **智能手势引导新手教程**：首次进入自动弹出分步动画教程，逐个演示绘画/擦除/换色手势，实时检测识别正确后才进入画布。

### 2.2 非功能需求

- **实时性**：手势识别与绘画延迟控制在 100ms 以内，画面流畅无卡顿；

- **稳定性**：长时间运行无崩溃、内存无泄漏，手势识别稳定无误判；

- **兼容性**：支持主流现代浏览器，跨终端正常使用；

- **安全性**：前端本地处理视频流，不上传原始画面，保护用户隐私；

- **可拓展性**：模块化开发，支持后续新增手势功能、用户登录、协同绘画等功能；

- **沉浸感**：落笔拖尾光效、线条发光渐变、实时音效反馈（落笔/擦除/换色/清空/完成临摹专属音效），移动端触觉震动适配；

- **传播性**：分享海报与短视频生成速度 < 3 秒，导出内容自带 Air Doodle 品牌标识与文案。

## 3 系统总体架构设计

### 3.1 整体架构

本系统采用**分层软硬件架构 + Next.js 前后端一体化架构**，整体分为五层，完全实现"硬件采集-前端计算-后端存储-页面展示"的完整闭环。

1. **硬件采集层**：设备摄像头通过 WebRTC API 采集实时视频流，将光学图像转化为数字像素帧，为系统提供原始数据输入。

2. **前端预处理层**：Next.js 客户端组件处理视频帧，完成画面翻转、缩放、画质优化，适配 AI 识别尺寸。

3. **Web AI 识别层**：MediaPipe Hands WASM 模型在浏览器本地解析手部 21 个关键点坐标，识别各类自定义手势（单指绘画、握拳暂停、手掌擦除、OK 切换、比耶贴纸、双手比心等），支持多手同时检测。

4. **交互渲染层**：HTML5 Canvas 分层渲染视频画面、手绘轨迹、手势关键点、动态背景、半透明临摹模板、特效粒子动画，实现实时涂鸦交互。

5. **全栈服务层**：Next.js API Route 提供文件上传、数据增删改查、用户管理、社区互动接口，Prisma ORM 操作数据库，服务端组件实现画廊 SSR 渲染。

### 3.2 技术栈详情

#### 前端技术栈（客户端）

- 核心框架：Next.js 15 (App Router)

- UI 框架：React 18 + Tailwind CSS（柔和渐变配色，童趣潮流卡通风格）

- 视频采集：WebRTC MediaDevices API

- AI 手势识别：MediaPipe Hands WASM（支持多手检测）

- 画布渲染：HTML5 Canvas 2D（分层渲染）

- 图片处理：html-to-image

- 视频录制：MediaRecorder API

- 音效系统：Web Audio API

#### 后端技术栈（服务端）

- 接口服务：Next.js API Routes

- 数据库 ORM：Prisma

- 数据库：SQLite（开发）/ PostgreSQL（生产）

- 文件存储：服务器本地文件系统

## 4 数据库设计

### 4.1 数据库选型

开发环境采用 SQLite 轻量数据库，无需额外安装服务，部署便捷；生产环境可无缝切换为 PostgreSQL，适配高并发场景。使用 Prisma ORM 统一管理数据模型与数据库操作，代码解耦、易于维护。

### 4.2 数据模型设计

在原有单一作品表基础上扩展，新增用户、成就、点赞模型，支持社区互动与成就系统，改动量小、兼容原有逻辑。

```Prisma
// ==================== 用户模型（新增） ====================
model User {
  id           String        @id @default(cuid())
  nickname     String?
  avatarUrl    String?       // 头像URL
  createdAt    DateTime      @default(now())
  achievements Achievement[] // 用户成就列表
  artworks     Artwork[]     // 用户作品列表
}

// ==================== 成就模型（新增） ====================
model Achievement {
  id          String   @id @default(cuid())
  name        String   // 成就名称（如"连续打卡3天""画笔大师"）
  desc        String   // 成就描述
  iconUrl     String?  // 成就图标URL
  unlockedAt  DateTime @default(now())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
}

// ==================== 作品模型（扩展） ====================
model Artwork {
  id           String   @id @default(cuid())
  imgUrl       String   // 作品图片存储路径
  name         String?  // 作品自定义名称
  isPublic     Boolean  @default(true)  // 是否公开投稿社区（新增）
  templateId   String?  // 临摹模板来源ID（新增）
  filterType   String?  // AI滤镜类型（新增）
  likesCount   Int      @default(0)     // 点赞数（新增）
  userId       String?  // 创作者用户ID（新增，可选兼容匿名）
  user         User?    @relation(fields: [userId], references: [id])
  createdAt    DateTime @default(now())
  likes        Like[]   // 点赞记录（新增）
}

// ==================== 点赞模型（新增） ====================
model Like {
  id        String   @id @default(cuid())
  artworkId String
  userId    String
  createdAt DateTime @default(now())
  artwork   Artwork  @relation(fields: [artworkId], references: [id], onDelete: Cascade)

  @@unique([artworkId, userId])  // 同一用户对同一作品仅可点赞一次
}
```

### 4.3 数据表字段说明

#### Artwork 表（扩展字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| isPublic | Boolean | 是否公开投稿社区，默认 true |
| templateId | String? | 临摹模板来源 ID，为空则为自由创作 |
| filterType | String? | 应用的 AI 滤镜类型（anime/oil/sketch/film） |
| likesCount | Int | 点赞数冗余字段，提升查询性能 |
| userId | String? | 创作者用户 ID，为空则为匿名创作 |

#### Achievement 表

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 成就名称 |
| desc | String | 成就描述 |
| iconUrl | String? | 成就图标 URL |
| unlockedAt | DateTime | 解锁时间 |

#### Like 表

| 字段 | 类型 | 说明 |
|------|------|------|
| artworkId | String | 被点赞作品 ID |
| userId | String | 点赞用户 ID |
| @@unique | - | 联合唯一约束，防止重复点赞 |

## 5 核心模块详细设计与实现

### 5.1 摄像头采集模块

该模块为系统硬件交互入口，仅在 Next.js 客户端组件运行，通过 WebRTC API 获取用户摄像头权限，实时拉取视频流。对画面进行水平镜像翻转，适配人体体感操作习惯，同时压缩视频分辨率，降低 AI 识别算力消耗。模块包含权限异常捕获，针对无摄像头、用户拒绝权限等场景给出友好提示。

### 5.2 前端 AI 手势识别模块

基于 MediaPipe Hands WASM 轻量化模型，浏览器离线加载模型，本地完成手部检测。单帧图像解析 21 个手部关键点三维坐标，通过计算指尖与指根的距离、手指开合角度，精准判断多种核心手势：食指单指、握拳、全掌摊开、左手 OK、右手 OK、双手握拳、双手比心、比耶手势等。MediaPipe 原生支持多手检测，无需更换模型即可实现双人同屏手势识别，通过区分左右手坐标实现双人独立手势判定。识别过程无需网络、无需后端参与，响应速度极快，彻底避免视频数据上传的隐私风险。

### 5.3 Canvas 涂鸦渲染模块

采用**多层分层 Canvas 渲染机制**：
- **底层**：动态背景层（星空、云朵、海浪、渐变等）+ 摄像头预览画面；
- **中层**：持久化涂鸦图层 + 半透明临摹模板叠加层 + 贴纸层；
- **上层**：手势关键点标注图层 + 特效粒子动画层（落笔拖尾光效、线条发光渐变）。

系统实时记录食指指尖坐标轨迹，连续坐标拟合平滑线条，实现手绘效果。根据手势状态动态切换画笔模式（普通画笔/特效画笔）、橡皮擦模式（局部擦除/雾化擦除）、暂停模式，支持一键清空画布（触发全屏烟花动画）。所有渲染操作均在客户端完成，交互实时性极强。

### 5.4 特效画笔系统

在原有纯色画笔基础上，新增体感动态笔刷系统：

- **星光笔**：沿轨迹生成闪烁星点粒子；
- **彩虹笔**：线条颜色随轨迹动态渐变七彩；
- **烟花笔**：落笔点触发微型烟花粒子扩散；
- **水彩晕染**：线条边缘柔和模糊，模拟水彩扩散；
- **荧光霓虹**：线条带发光光晕效果；
- **像素方块**：轨迹离散为像素风格方块；
- **爱心画笔**：双手比心手势触发，轨迹自动点缀爱心图案。

不同 OK 手势组合切换特效画笔，画笔状态在侧边手势可视化面板实时显示。

### 5.5 体感小游戏模块

复用手部关键点识别逻辑，开发轻量化内置小游戏，一键切换画布模式：

1. **手势消消画**：Canvas 随机生成彩色圆点网格，检测食指指尖轨迹连线，判断相同颜色相邻圆点，连线成功后消除并加分，支持关卡递进；
2. **隔空抓星星**：requestAnimationFrame 循环随机生成星星粒子，检测手掌开合状态（五指摊开→握拳视为"抓取"），判断星星坐标是否在手掌区域内，收集计分，限时闯关；
3. **手势猜谜画画**：系统从词库随机抽取词语，启动倒计时，用户在 Canvas 上手绘该词语，计时结束后通过前端轻量图像匹配或人工评分机制给出分数。

### 5.6 社交分享模块

- **录屏导出**：前端 MediaRecorder API 录制 Canvas 画面，生成 3-10 秒 WebM/MP4 短视频；
- **分享海报生成**：将画作截图与预设趣味边框、文案模板（"我用隔空手势画出XX"）、Air Doodle 专属水印合成分享海报；
- **一键分享**：支持下载图片/视频到本地、复制分享链接、调用 Web Share API 分享到支持的平台；
- **作品社区**：公开作品支持点赞、收藏、评论，排行榜按今日热门、本周最佳、创意维度分类展示。

### 5.7 Next.js 后端接口模块

#### 5.7.1 作品上传接口

接收前端 FormData 格式的图片文件与作品名称，通过 UUID 生成唯一文件名，保存至服务器静态资源目录，同时通过 Prisma 向数据库写入作品信息（含公开状态、模板来源、滤镜类型），返回作品存储结果，完成数据持久化。

#### 5.7.2 作品查询接口

支持分页查询所有公开作品数据，按创作时间/点赞数排序，支持按模板类别、滤镜类型筛选，为画廊页面与社区页面提供数据支撑，适配服务端 SSR 渲染。

#### 5.7.3 作品删除接口

根据作品 ID，同步删除数据库记录与服务器存储的图片文件，保证数据与资源一致性。

#### 5.7.4 社区互动接口（新增）

- **点赞接口**：用户对作品点赞/取消点赞，事务更新 likesCount 冗余字段；
- **评论接口**：作品评论的增删查；
- **排行榜接口**：按时间范围与排序维度查询热门作品。

#### 5.7.5 成就接口（新增）

- **成就查询**：查询当前用户已解锁成就列表；
- **成就解锁**：满足条件时自动写入成就记录，前端弹窗提示。

### 5.8 画廊 SSR 展示模块

画廊页面采用 Next.js 服务端组件开发，页面首次加载时服务端直接查询数据库，预渲染所有作品卡片，无需前端异步请求，页面加载速度快、SEO 友好。社区页面扩展为互动模式，支持点赞、评论、排行榜切换。支持响应式布局，自动适配电脑、手机屏幕尺寸。

## 6 核心功能逻辑说明

### 6.1 基础手势交互

- **食指伸直涂鸦**：识别单根食指伸直、其余手指弯曲，持续记录指尖坐标并连线，生成手绘线条。
- **握拳暂停绘画**：五指全部弯曲握拳，锁定绘画状态，手部移动不生成笔迹。
- **手掌擦除**：五指完全摊开，以手掌区域为范围，局部清除画布涂鸦内容；双手同时摊开触发雾化擦除动画。
- **OK 手势换色**：左手 OK 切换红色画笔，右手 OK 切换蓝色画笔，双手 OK 切换绿色画笔。
- **双手握拳清空画布**：双手握拳保持 2 秒，触发全屏烟花动画，自动清空整张画布所有涂鸦。

### 6.2 模板临摹手势

- **切换模板**：握拳切换上一张模板，摊手切换下一张模板；
- **调节透明度**：OK 手势配合上下移动调整模板透明度；
- **完成判定**：描线覆盖率超过阈值自动弹出完成勋章。

### 6.3 特效画笔切换

- **切换特效**：单手 OK + 手指数量组合切换特效画笔（星光/彩虹/烟花/水彩/霓虹/像素）；
- **爱心画笔**：双手比心（双手中指无名指弯曲，拇指食指构成心形）自动激活爱心画笔模式。

### 6.4 贴纸操作手势

- **呼出面板**：比耶手势（食指中指伸直张开）唤出/关闭贴纸面板；
- **拖拽放置**：单指选中贴纸并拖拽到目标位置；
- **缩放**：握拳状态下手部前后移动控制贴纸大小；
- **删除**：手掌覆盖贴纸区域擦除贴纸。

### 6.5 双人同屏模式

- **左右区分**：根据手部关键点 X 坐标中值与画面中线对比，区分左右侧用户；
- **独立操作**：左右侧手势独立判定，互不干扰，支持双人同时绘画或对战小游戏。

### 6.6 全栈业务流程

- **作品云端保存**：用户点击保存，前端将 Canvas 画面转为 PNG 图片，调用后端接口完成上传与存储，可选公开/私密；
- **在线画廊浏览**：服务端渲染所有公开作品，支持在线预览、点赞、评论、复刻、管理、删除；
- **成就解锁**：后端检测用户行为数据，满足条件自动写入成就记录，前端实时弹窗提示。

## 7 系统部署与运行说明

### 7.1 环境安装步骤

1. 安装 Node.js 18+ 环境；

2. 执行 npm install 安装所有项目依赖；

3. 执行 npx prisma migrate dev --name init 初始化数据库；

4. 创建 public/uploads、public/templates、public/stickers、public/audio 目录。

### 7.2 本地运行

执行 npm run dev，访问 http://localhost:3000 即可进入项目主页，开启摄像头权限后即可使用手势涂鸦功能。

### 7.3 线上部署

项目支持 Vercel 一键零配置部署，也可通过服务器打包部署。线上环境必须配置 HTTPS 证书，浏览器仅允许 HTTPS/localhost 域名调用摄像头权限。执行 npm run build && npm start 完成生产环境部署。
